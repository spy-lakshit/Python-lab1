1. Project Title
Placement Drive Eligibility & Merit Ranking System

2. Problem Statement
During campus placements, companies conduct aptitude tests. The TPO (Training and Placement Cell) receives a raw text file of student scores. They need a tool to automatically read these scores, filter out students who failed the cutoff, and generate a ranked Merit List based on their marks.

3. Objective
To read numerical data from an external file, process it using user-defined functions, and apply the Selection Sort algorithm to rank students based on their performance.

4. Python Concepts Used
Sorting Algorithms: Selection Sort (Sorting in Descending Order).

File Handling: Reading raw data, writing processed data to a new file.

Lists and Tuples: Storing data as lists of sub-lists [[name, score], ...].

Type Casting: Converting string scores to integers for mathematical comparison.

5. Data Handling (File Structure)
Input File: aptitude_results.txt (Needs to be created manually before running).
Format:

Output File: merit_list.txt (Generated by Python).

6. Complete Python Code

def read_scores(filename):
    """Reads the raw score file and returns a list of [name, score]"""
    student_data = []
    try:
        with open(filename, 'r') as file:
            for line in file:
                line = line.strip()
                if line:
                    parts = line.split(',')
                    name = parts[0].strip()
                    # Type casting string to integer for sorting
                    score = int(parts[1].strip()) 
                    student_data.append([name, score])
    except FileNotFoundError:
        print(f"Error: {filename} not found! Please create it first.")
    return student_data

def selection_sort_descending(arr):
    """Sorts the 2D list based on scores (index 1) in descending order."""
    n = len(arr)
    for i in range(n):
        max_idx = i
        for j in range(i + 1, n):
            # Comparing the scores (arr[j][1])
            if arr[j][1] > arr[max_idx][1]:
                max_idx = j
        
        # Swap the elements
        arr[i], arr[max_idx] = arr[max_idx], arr[i]
    return arr

def generate_merit_list():
    input_file = "aptitude_results.txt"
    output_file = "merit_list.txt"
    cutoff_marks = 60
    
    # Step 1: Read Data
    print("Reading raw aptitude scores...")
    data = read_scores(input_file)
    if not data:
        return

    # Step 2: Filter Data (Remove scores below cutoff)
    eligible_students = []
    for student in data:
        if student[1] >= cutoff_marks:
            eligible_students.append(student)

    # Step 3: Sort Data using Selection Sort
    print("Ranking students using Selection Sort...")
    sorted_students = selection_sort_descending(eligible_students)

    # Step 4: Write to new file and display
    print("\n--- Final Merit List ---")
    with open(output_file, 'w') as file:
        file.write("Rank | Name | Score\n")
        file.write("-" * 25 + "\n")
        
        rank = 1
        for student in sorted_students:
            record = f"{rank}. {student[0]} - {student[1]}"
            print(record)
            file.write(record + "\n")
            rank += 1
            
    print(f"\nSuccess! Merit list saved to '{output_file}'.")

# Run the program
if __name__ == "__main__":
    generate_merit_list()
7. Step-by-Step Explanation
Reading: The program opens the text file, splits the line by commas, and crucially, uses int() to convert the text score into a number so we can do math on it.

Filtering: It loops through the list and appends only those students to a new list who scored >= 60.

Sorting: It passes the filtered list to our custom selection_sort_descending() function. Selection sort repeatedly finds the highest score from the unsorted portion and moves it to the front.

Writing: It loops through the newly sorted list, assigns a rank counter (rank += 1), prints it to the screen, and writes it to merit_list.txt.

9. Algorithm Explanation
Selection Sort Time Complexity: O(n²) (where n is the number of students). We have two nested for loops. While not the fastest algorithm in the world, it is highly memory efficient (Space Complexity O(1)) because it sorts the array "in-place" without requiring extra lists.

10. Viva Questions & Answers
Why did you use int() while reading the file?

Answer: Data read from a text file is always in String format. If I don't typecast it to an integer, Python will sort them alphabetically instead of numerically (e.g., "9" would be considered greater than "100").

Explain how your Selection Sort function works.

Answer: It uses two nested loops. The outer loop tracks the current position. The inner loop scans the rest of the list to find the absolute highest score. Once found, it swaps that highest score with the current position.

Why did you compare arr[j][1] > arr[max_idx][1] instead of just arr[j] > arr[max_idx]?

Answer: My list is a 2D list containing [Name, Score]. Index 0 is the name, and index 1 is the score. I specifically need to compare index 1 to rank them by marks, not by alphabetical names.

What is the Time Complexity of Selection Sort?

Answer: The time complexity is O(n²) for all cases (worst, average, and best) because the two nested loops will always iterate through the array regardless of whether it is already sorted or not.

What happens in your code if the aptitude_results.txt file does not exist?

Answer: I used a try-except block to catch the FileNotFoundError. Instead of crashing the whole program with a red error, it gracefully prints a warning telling the user to create the file first.
